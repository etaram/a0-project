# הכנה לבדיקות SoundMaster - גרסה 1.0

## הכנת הסביבה לבדיקות

### הכנת סביבת פיתוח
1. **הכנת בילד לבדיקות**:
   ```bash
   # עדכון גרסה בקובץ app.json
   expo build:android -t apk --release-channel beta
   
   # או בניית האפליקציה עם EAS
   eas build -p android --profile preview
   ```

2. **הכנת סביבת בדיקות**:
   - הכן רשימת מכשירי בדיקה מייצגים:
     * לפחות מכשיר אחד מכל גרסת אנדרואיד (9-13)
     * מכשירים מיצרנים שונים (Samsung, Xiaomi, Google)
     * גדלי מסך שונים (קטן, רגיל, גדול)
   - הכן אביזרים לבדיקה:
     * אוזניות חוטיות
     * מספר מכשירי Bluetooth שונים
     * רמקולים חיצוניים (אם רלוונטי)

3. **הכנת מערכת לניהול בדיקות**:
   - הקם טבלת מעקב בדיקות (Google Sheets או Trello)
   - הכן תבנית לדיווח באגים
   - הגדר צוות בודקים והקצה תחומי בדיקה

### הכנת מכשירי בדיקה
1. **התקנת האפליקציה**:
   ```bash
   # התקנה על מכשיר פיזי מחובר
   adb install -r ./soundmaster-beta.apk
   
   # התקנה על כל המכשירים המחוברים
   adb devices | tail -n +2 | cut -sf 1 | xargs -I X adb -s X install -r ./soundmaster-beta.apk
   ```

2. **הגדרה לפני בדיקות**:
   - נקה היסטוריית נתוני אפליקציה
   - כבה את כל אפליקציות השמע האחרות
   - וודא שיש מספיק סוללה (לפחות 50%)
   - צלם צילומי מסך לפני הבדיקה

3. **הכנת אפליקציות נוספות**:
   - התקן אפליקציות שמע פופולריות לבדיקה:
     * Spotify או שירות מוזיקה אחר
     * YouTube או שירות וידאו אחר
     * נגן פודקאסטים
     * אפליקציית שיחות (WhatsApp, Telegram)
     * משחק קל עם שמע

## תוכנית בדיקות מפורטת

### 1. בדיקת התקנה והרשאות
- **בדיקת ההתקנה**:
  * התקנה חדשה
  * התקנה מעל גרסה קודמת (שדרוג)
  * הסרה והתקנה מחדש

- **בדיקת הרשאות**:
  * בדיקה ראשונית עם הרשאות מלאות
  * בדיקה עם דחיית הרשאות ואישור מאוחר יותר
  * בדיקה עם דחיית הרשאות לצמיתות ואיפוס דרך הגדרות

### 2. בדיקות פונקציונליות מורחבות

- **בדיקת זיהוי מקורות שמע**:
  * **תרחיש מינימלי**: מקור שמע יחיד
    - הפעל Spotify בלבד ובדוק אם מזוהה
  * **תרחיש סטנדרטי**: 2-3 מקורות שמע
    - הפעל Spotify + YouTube ובדוק זיהוי נכון
  * **עומס מקסימלי**: מקסימום מקורות שמע
    - הפעל 5+ אפליקציות שמע בו זמנית ובדוק את ביצועי המערכת

- **בדיקת שליטה בווליום**:
  * **בקרת ווליום בסיסית**:
    - שנה ווליום ב-10% קפיצות
    - ודא שהשינוי משפיע רק על אפליקציה ספציפית
  * **שינויי ווליום מהירים**:
    - החלק את הסליידר במהירות מינימום למקסימום ובחזרה
    - בדוק תגובה ועקביות
  * **השתקה**:
    - בדוק השתקה מלאה (ווליום 0)
    - ודא שהממשק מתעדכן בהתאם

- **בדיקת יציאות שמע**:
  * **החלפת יציאה**:
    - בדוק מעבר בין כל היציאות (רמקול > אוזניות > בלוטות')
    - ודא שהשמע יוצא מהיציאה הנכונה פיזית
  * **מניעת התנגשויות**:
    - נסה להשמיע שני מקורות באותה יציאה
    - ודא שמוצגת התראה מתאימה
  * **אכיפה**:
    - בדוק אפשרות אכיפה להחלפת יציאת שמע
    - ודא שמקור קודם נעצר

- **בדיקת Quick Settings**:
  * התקנה וההגדרה של ה-tile
  * פונקציונליות בלחיצה קצרה
  * פונקציונליות בלחיצה ארוכה

### 3. בדיקות עמידות ומקרי קצה

- **מצבי חומרה**:
  | בדיקה | צעדים | תוצאה מצופה |
  |-------|--------|-------------|
  | חיבור/ניתוק אוזניות | חבר/נתק אוזניות תוך שימוש באפליקציה | האפליקציה מתעדכנת, אין קריסות |
  | הפעלת בלוטות' | הפעל/כבה בלוטות' בזמן שימוש | האפליקציה מתעדכנת ללא קריסות |
  | מצב טיסה | הפעל/כבה מצב טיסה | האפליקציה ממשיכה לתפקד עם מקורות מקומיים |
  | סוללה נמוכה | גרום לסוללה נמוכה (דרך סימולציה) | האפליקציה עוברת למצב חיסכון בסוללה |

- **מצבי תוכנה**:
  * אתחול מחדש של המכשיר
  * שיחה נכנסת בזמן שימוש
  * התראה דחופה בזמן שימוש
  * שינוי שפה במכשיר
  * שינוי גודל גופן מערכת

### 4. בדיקות ביצועים וחיסכון בסוללה

- **בדיקות ביצועים**:
  * מדידת זמן תגובה לשינויי ווליום
  * מדידת זמן תגובה להחלפת יציאת שמע
  * זמן הפעלה ראשוני של האפליקציה

- **בדיקות סוללה**:
  * צריכת סוללה במצב פעיל למשך שעה
  * צריכת סוללה במצב רקע למשך שעה
  * השוואה למצב ללא האפליקציה

### 5. בדיקות מערכת שאלונים

הכן סקר בדיקות עם השאלות הבאות:
1. דירוג קלות השימוש (1-10)
2. דירוג העיצוב (1-10)
3. תכונה אהובה ביותר
4. תכונה שהכי זקוקה לשיפור
5. האם האפליקציה השיגה את המטרה העיקרית שלה?
6. האם נתקלת בבעיות? אם כן, אילו?
7. הצעות לשיפור

## הערכות להשקה

- **קריטריונים להשקה**:
  * אפס באגים קריטיים
  * פחות מ-5 באגים בעדיפות גבוהה
  * כל תכונות הליבה פועלות ב-95% מהמכשירים הנתמכים
  * דירוג ממוצע של 4/5 בסקרי הבטא

- **תהליך דיווח באגים**:
  1. צלם מסך/וידאו של הבאג
  2. תעד את הצעדים לשחזור
  3. ציין את מודל המכשיר וגרסת האנדרואיד
  4. שלח דרך טופס הבדיקות או למייל bug-reports@soundmaster-app.com

- **תהליך דחיפת תיקונים**:
  * תיקון באגים קריטיים תוך 24 שעות
  * עדכוני גרסה מתוכננים פעם בשבוע במהלך הבטא
  * רשימת שינויים מעודכנת עם כל גרסה
